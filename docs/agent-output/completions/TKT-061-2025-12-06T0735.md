### Notes

The incident response section added provides:

1. **Quick Response Guide** - A table format for rapid triage of the 4 main service types
2. **Detailed Runbooks** - Comprehensive guides for each monitored service:
   - Dashboard (greetnow.com on Vercel)
   - Signaling Server (Railway)
   - Supabase API
   - Widget CDN
3. **Escalation Policy** - Clear criteria for when to investigate vs. page, plus 3-level escalation chain
4. **Communication Templates** - Ready-to-use templates for Slack and customer communications
5. **Post-Incident Checklist** - Ensures proper incident closure and learning
6. **Common False Alarms** - Helps avoid panic over benign alerts
7. **Emergency Contacts** - Quick reference for who to contact

Each service runbook includes:
- Common causes (most likely failure modes)
- Diagnostic steps (bash commands to verify the issue)
- Resolution steps (prioritized by likelihood)
- Who can fix (role/access level needed)
- When to escalate (clear time/severity thresholds)

This resolves the core issue from F-647: when someone gets paged at 3 AM, they now have a clear, actionable playbook to follow instead of just a phone number to call.
